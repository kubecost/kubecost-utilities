# Kubecost Object Store Cleaner

Kubecost stores files that represent a window of time. There are three different resolutions: 10m, 1h, 1d

Only the 1d resolution files are required during a rebuild of the Aggregator pod- which would be needed if the Primary is moved to a new cluster or the PV was lost.

The 10m and 1h files are not required for a rebuild, but they will be ingested up to the limit set in the helm values, which defaults to `retention1h: 49` (49 hours) and `retention10m: 36` (360 minutes)

**These defaults may change in the future**

The [script](kubecost-object-store-cleaner.sh) uses a conservative default of removing files after 7 days.

- **Two Operation Modes:**
  - The default mode is readonly: search S3 and generate a CSV of files older than 7 days
  - `--delete` will delete the files

## Example Usage

Generate a csv of the 10m and 1h files older than 7 days:

```sh
export BUCKET=kubecost-federated-s3-bucket
./object-storage-cleanup/kubecost-object-store-cleaner.sh
```

Delete the files generated by the csv:

```sh
./object-storage-cleanup/kubecost-object-store-cleaner.sh --delete --csv=old_files.csv
```

Just delete, do not output a list of files

```sh
./object-storage-cleanup/kubecost-object-store-cleaner.sh --delete
```

## Prerequisites

- AWS CLI configured with appropriate credentials
- S3 bucket access permissions (read for query, delete for cleanup)

## Configuration

Set via environment variables or modify script defaults:

```bash
BUCKET="your-bucket-name"           # S3 bucket name
PREFIX="federated/"                 # S3 prefix to search (ignored if SEARCH_PATTERNS set)
DAYS_OLD="7"                        # Files older than N days
OUTPUT_FILE="old_files.csv"         # Output CSV filename
SEARCH_PATTERNS="1h,10m"            # Comma-separated patterns to match in file paths
```

## More Example Usage

### Mode 1: Query S3 and Generate CSV

Find files matching patterns older than 7 days:
```bash
BUCKET=my-kubecost-bucket SEARCH_PATTERNS="1h,10m" ./kubecost-object-store-cleaner.sh
```

Find all files in prefix older than 30 days:
```bash
BUCKET=my-kubecost-bucket PREFIX="federated/" SEARCH_PATTERNS="" DAYS_OLD=30 ./kubecost-object-store-cleaner.sh
```

### Mode 2: Delete Files

**Option A: Query and delete in one step (with confirmation):**
```bash
./kubecost-object-store-cleaner.sh --delete
```
This will query S3, show preview, and prompt for confirmation before deleting.

**Option B: Delete from existing CSV:**
```bash
./kubecost-object-store-cleaner.sh --delete --csv=old_files.csv
```

## Safety Features

- **Preview First:** Always shows preview before deletion
- **Confirmation Prompt:** Requires "yes" confirmation when using --delete flag
- **CSV Audit Trail:** Maintains record of what will be/was deleted
- **No Wildcards:** Deletes only files explicitly listed in CSV

## Output Format

### Query Output (old_files.csv)
CSV with three columns:
```csv
Key,LastModified,Size
federated/cluster1/1h/2026-03-01.json,2026-03-01T00:00:00,1024
federated/cluster1/10m/2026-03-01.json,2026-03-01T00:00:00,512
```

### Deletion Log (old_files_deleted.csv)
Automatically created during deletion with four columns:
```csv
Key,LastModified,Size,DeletedAt
federated/cluster1/1h/2026-03-01.json,2026-03-01T00:00:00,1024,2026-04-16T11:21:00Z
federated/cluster1/10m/2026-03-01.json,2026-03-01T00:00:00,512,2026-04-16T11:21:00Z
```

## Performance

- **Batch Deletion:** Uses `aws s3api delete-objects` to delete up to 1000 objects per API call
- **Speed:** ~100-1000x faster than sequential deletion for large file counts
- **Example:** Deleting 10,000 files takes ~10-20 seconds instead of 30+ minutes
- **Deletion Logging:** Automatically creates audit trail of deleted files

## Notes

- **SEARCH_PATTERNS** takes precedence over **PREFIX** - set `SEARCH_PATTERNS=""` to use prefix-based search
- Dates are in UTC
- File size is in bytes
- Script uses `aws s3api` for querying and batch deletion
